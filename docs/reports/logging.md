# レポート: ロギング導入 (`logging`)

## 実施概要

- アプリケーション全体にロギング基盤を導入し、主要なイベント・エラーが記録されるようにした。
- `main.py` にlogging設定を追加し、コンソールとファイル（`app.log`）の両方にログを出力するようにした。
- 各モジュール（`config.py`, `task_manager.py`, UIモジュール）でロガーを使用し、適切なログレベルでログ出力を実装した。

## 実装内容

### 1. ロギング設定（`main.py`）

- `logging.basicConfig()` でログレベル、フォーマット、ハンドラーを設定
- ログレベル: `INFO`（デフォルト）
- 出力先: コンソール（標準出力）とファイル（`app.log`）
- ファイル出力は追記モード（実行のたびに追記される）

### 2. 各モジュールでのログ出力

#### `config.py`
- ファイル読み書きの成功/失敗をログ出力
- 成功時: `INFO` レベル
- エラー時: `ERROR` レベル（スタックトレースも記録）

#### `task_manager.py`
- タスク監視の開始/停止を `INFO` レベルで記録
- 通知発火を `DEBUG` レベルで記録
- タスク完了/未完了の記録を `INFO` レベルで記録
- エラーハンドリングで `print()` を `logger.error()` に置き換え

#### UIモジュール
- `ui/main_window.py`: 設定画面・ログ画面表示時のエラーログ
- `ui/settings_window.py`: タスク追加/編集/削除時のエラーログ
- `ui/reminder_window.py`: タスク完了処理時のエラーログ
- `ui/log_window.py`: 表示更新時のエラーログ

### 3. ログレベル

- **DEBUG**: 詳細なデバッグ情報（通知発火など）
- **INFO**: 一般的な情報（タスク保存、監視開始など）
- **ERROR**: エラー（処理失敗、スタックトレースも記録）

## メモ

- ログファイル（`app.log`）はプロジェクトルートに作成される。
- ログは実行のたびに追記されるため、過去のログも確認可能。
- エラー発生時は `exc_info=True` でスタックトレースも記録されるため、トラブルシューティングが容易になる。
- ログレベルを `DEBUG` に変更すると、より詳細な情報が得られる。
